# todo-frontend — 需求规格

## 概述

移动端 Todo List 页面，支持任务的查看、新增、完成切换、删除和状态筛选。

## 项目信息

- 项目名: my-todo-list
- 架构类型: turborepo-monorepo
- 涉及层: 前端层（apps/web）

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-18 | v1   | 初始需求 |

## 用户故事

- 作为普通用户（无需登录），我想在手机上管理我的待办任务，以便快速记录和追踪日常事项。

## 功能需求

1. [F-001] 页面加载时从后端获取任务列表，展示加载、空、错误三种状态
2. [F-002] 用户可在输入框输入标题并提交，创建新的未完成任务
3. [F-003] 用户可点击勾选控件切换任务完成/未完成状态
4. [F-004] 用户可点击删除按钮，经底部确认弹层确认后删除任务
5. [F-005] 用户可通过「全部 / 未完成 / 已完成」Tab 在前端过滤列表
6. [F-006] 页面适配移动端，所有交互控件尺寸适合手指触控

## 非功能需求

- 性能: 列表更新使用 TanStack Query 的 invalidateQueries，不刷新整页
- 体验: 操作失败时通过 sonner toast 提示，不阻断页面
- 适配: 仅适配手机端视口，不要求桌面端布局

## 验收标准

- [ ] [AC-001] 页面加载时展示加载状态，数据返回后展示任务列表
- [ ] [AC-002] 接口失败时展示错误提示，提供重新加载按钮
- [ ] [AC-003] 列表为空时展示对应空状态文案（区分筛选场景）
- [ ] [AC-004] 输入有效标题后点击「添加」，任务出现在列表中，输入框清空
- [ ] [AC-005] 输入为空时点击「添加」，不触发请求
- [ ] [AC-006] 点击勾选控件，任务完成状态切换，视觉上有明显区分（删除线或弱化）
- [ ] [AC-007] 点击删除按钮弹出底部确认弹层，点取消不删除，点确认删除后任务消失
- [ ] [AC-008] 点击筛选 Tab 后，列表即时更新，当前 Tab 有高亮
- [ ] [AC-009] 页面标题区展示未完成任务数统计
- [ ] [AC-010] 所有按钮、输入框高度适合手指点击（建议 min-h-11）
